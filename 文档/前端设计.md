# 前端设计 (Vue 3)

## 1. 技术架构
- **核心框架**: Vue 3 (Composition API)
- **构建工具**: Vite
- **路由管理**: Vue Router 4
- **状态管理**: 响应式 API (ref/reactive) + EventBus (简单场景)
- **UI 组件库**: Element Plus
- **HTTP 客户端**: Axios
- **图表库**: ECharts

## 2. 目录结构 (`frontend/src`)
- **api**: 接口封装
    - 按模块拆分 (`auth.js`, `products.js`, `orders.js` 等)，统一管理 API 请求。
    - `http.js`: Axios 实例配置，统一处理 BaseURL、请求拦截（注入 Token）、响应拦截（全局错误处理、401 跳转）。
- **views**: 页面视图
    - **admin**: 后台管理页面 (`AdminLayout`, `AdminDashboard`, `AdminProducts`, `AdminOrders` 等)。
    - **User Pages**:
        - `HomeView.vue`: 首页（轮播、热销、推荐）。
        - `ProductListView.vue`: 商品列表（搜索、筛选）。
        - `ProductDetailView.vue`: 商品详情（图集、加购）。
        - `CartView.vue`: 购物车（数量调整、结算）。
        - `OrderSubmitView.vue`: 订单提交（选择地址）。
        - `OrderListView.vue` / `OrderDetailView.vue`: 订单管理。
        - `UserCenterView.vue`: 个人中心。
        - `AuthView.vue`: 登录/注册页。
- **router**: 路由配置
    - `index.js`: 定义路由规则，配置全局前置守卫 (`beforeEach`) 处理登录校验 (`meta.requiresAuth`) 和管理员权限校验 (`meta.requiresAdmin`)。

## 3. 核心交互流程
- **认证流程**: 登录成功 -> 存储 Token 到 localStorage -> 跳转回原页面。
- **购物流程**: 浏览商品 -> 加入购物车 (API) -> 购物车页面修改数量 -> 结算 -> 创建订单 -> 支付(模拟) -> 查看订单。
- **管理流程**: 管理员登录 -> 仪表盘查看数据 -> 商品/订单管理 -> 增删改查操作实时反馈。

## 4. 关键实现
- **路由守卫**: 防止未登录用户访问敏感页面，防止普通用户访问后台。
- **响应式布局**: 使用 Element Plus 的 Grid 系统适配不同屏幕。
- **组件复用**: 提取公共布局组件 (`AdminLayout`) 减少重复代码。
