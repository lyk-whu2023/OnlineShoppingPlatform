## 分类

- `GET /categories`
  - 响应：`CategoryDTO[]`
- `POST /categories`
  - 请求：`{ name }`
  - 响应：`CategoryDTO`
- `PUT /categories/{id}`、`DELETE /categories/{id}`

## 商品

- `GET /products`
  - 查询参数：`ProductQueryDTO`（`categoryId, keyword, minPrice, maxPrice, sort, page, size`）
  - `sort` 支持：`sales`（销量优先）、`price_asc`（价格升序）、`price_desc`（价格降序）
  - 响应：`{ list: ProductDTO[], total, page, size }`
- `GET /products/{id}`
  - 响应：`ProductDTO`
- `GET /products/{id}/images`
  - 响应：`string[]`（图片 URL 或种子）
- `POST /products`
  - 请求：`ProductDTO`（不含 `id`，`images` 可选）
  - 响应：`ProductDTO`
- `PUT /products/{id}`、`DELETE /products/{id}`

## 用户与认证

- `POST /auth/login`
  - 请求：`{ username, password }`
  - 响应：`{ token, user: UserDTO }`
- `POST /auth/register`
  - 请求：`{ username, password, phone }`
  - 响应：`{ token, user: UserDTO }`
- `POST /auth/logout`
  - 响应：`204`
- `GET /users/me`
  - 响应：`UserDTO`
- 管理端：
  - `GET /users`
  - `PATCH /users/{id}/status`（`{ status }`）

## 地址

- `GET /addresses`
  - 响应：`AddressDTO[]`
- `POST /addresses`
  - 请求：`AddressDTO`（不含 `id`）
  - 响应：`AddressDTO`
- `PUT /addresses/{id}`、`DELETE /addresses/{id}`

## 购物车

- `GET /cart`
  - 响应：`CartItemDTO[]`
- `POST /cart/items`
  - 请求：`CartItemDTO`
  - 响应：`CartItemDTO[]`
- `PUT /cart/items/{productId}`（更新数量）
  - 请求：`{ qty }`
  - 响应：`CartItemDTO[]`
- `DELETE /cart/items/{productId}`、`DELETE /cart`（清空）

## 路由与登录守卫

- 前端路由在以下页面启用登录守卫：`/product/:id`、`/cart`、`/order/submit`、`/user*`、`/order/:id`
- 未登录访问上述页面时，将重定向至 `/login`，并携带 `redirect=<原始路径>` 参数；登录/注册成功后自动返回原页面

## 订单

- `POST /orders`
  - 请求：`{ addressId }`（后端依据用户当前购物车创建）
  - 响应：`OrderDTO`
- `GET /orders/{id}`
  - 响应：`OrderDTO`
- `GET /orders`
  - 查询：`userId` 或 `status`
  - 响应：`OrderDTO[]`
- `PATCH /orders/{id}/status`
  - 请求：`{ status }`

## 支付

- `GET /payments/{orderId}`、`PUT /payments/{orderId}`
  - 请求：`PaymentDTO`
  - 响应：`PaymentDTO`

## 物流

- `GET /shipments/{orderId}`、`PUT /shipments/{orderId}`
  - 请求：`ShipmentDTO`
  - 响应：`ShipmentDTO`

## 认证与错误约定

- 请求头自动注入：`Authorization: Bearer <token>`（见 `frontend/src/api/http.js:3`），后端拦截器解析用户信息
- 统一错误格式：`{ code, message, details? }`
- 常见状态码：
  - `400` 参数错误、`401` 未认证、`403` 无权限、`404` 资源不存在
  - `409` 冲突（库存不足等）、`500` 服务内部错误
