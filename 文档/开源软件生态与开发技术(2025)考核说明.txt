武汉大学计算机学院 2025-2026 学年第一学期
《开源软件生态与开发技术》期末考核说明
年级： 2023 级                          学院：计算机学院 
专业方向：软件工程           			层次：本科 
本课程是一门选修课程，旨在通过实践深入理解开源软件生态体系，掌握主流开源技术与框架的应用。课程将帮助学生巩固面向对象程序设计基础，了解开源社区文化，学习使用主流开源技术栈进行B/S架构企业级应用开发，提高利用开源解决方案解决实际问题的能力。通过该实践课程，进一步加深对 Java 面向对象程序设计、Java 平台企业版（Java Platform Enterprise Edition）、企业级开发框架Spring的学习和掌握，要求能够使用JavaEE相关主流技术、框架和规范，开发B/S架构企业级应用软件。
要求对课程中所讲授内容的深入理解，巩固其他先修课程中所学的编程知识，同时提高利用编程思维分析问题、解决问题的能力。使用 Java 语言、JavaEE相关主流技术，选择一个B/S架构平台的应用场景，进行完整的需求分析、概要设计、详细设计（包括数据库设计、界面设计、算法设计等），并使用SSM框架加以开发实现。具体提交要求如下： 
1）期末实验（以个人为单位，独立完成）
	一份程序代码和相关资源
	每位成员单独完成的实验报告一份（武汉大学实验报告模板）
	系统的视频介绍（5-10分钟），介绍自己的工作，然后制作成完整的一个视频。需要介绍系统功能、特色、关键实现等内容。
2）课程的总评成绩分配比例如下：考勤(15)+平时作业(15)+课堂开源项目分享展示（15）+ 期末实验55 = 100
提交方式：课程QQ群中，以群作业的方式（按照课头和平时/期末）进行布置和提交，请大家认准提交位置。
截止时间：2025年12月19日		



考核标准：期末实验考核标准（占总成绩的55%）
期末实验以个人为单位，开发一个 B/S 架构的企业级应用软件。考核标准涵盖项目代码质量、实验报告、系统介绍视频等内容：
1.	项目代码与实现（占25%）：
	功能完整性（40%）
系统的功能需求是否完整实现，包括核心功能和次要功能。要求根据设计的需求文档实现完整的业务逻辑。
	代码规范与质量（30%）
代码是否符合 Java 编程规范，具备良好的可读性和维护性。要求代码结构清晰，模块划分合理，注释充分，避免硬编码和冗余。
	系统性能与稳定性（10%）
系统是否能够高效稳定运行，响应速度是否合理，是否能在测试中通过各种预期场景下的测试。
	技术创新性与复杂性（20%）
鼓励在项目中使用一些较为先进或复杂的技术（如复杂的数据库操作、接口设计、权限控制等），以提高项目的技术含量。

2.	实验报告（占20%）：
	结构与内容（50%）
实验报告需按规定格式撰写，包括项目的需求分析、设计思路、技术实现细节、数据库设计、测试情况等。
	分析与总结（30%）
报告需对项目开发过程中的遇到的问题、解决方案、项目中的关键点进行分析和总结，体现出对项目的深入思考。
	规范与格式（20%）
报告需符合武汉大学实验报告模板的要求，内容有序、条理清晰、格式规范。
3.	系统视频介绍（占10%）：
	内容展示（50%）
视频需完整展示系统的核心功能、特色及关键实现，重点突出项目的亮点和创新之处。
	视频制作质量（50%）
视频制作需清晰、有序，声音、字幕等清晰可辨，体现出合理的排版和展示逻辑。
